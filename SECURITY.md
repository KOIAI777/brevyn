# Security Policy

## Supported Code

Security fixes target the current `main` branch and the latest Community release when one exists.

## Reporting a Vulnerability

Do not open a public issue for suspected vulnerabilities, exposed credentials, unsafe file handling, or sandbox escapes. Use GitHub's private vulnerability reporting for this repository:

1. Open the repository's **Security** tab.
2. Choose **Report a vulnerability**.
3. Include affected versions, reproduction steps, impact, and any suggested mitigation.

This repository does not operate Brevyn Official accounts, billing, or hosted model services. Reports about those systems must be sent through the Official support channel rather than a public Community issue.
